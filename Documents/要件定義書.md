## 目的
> アプリ開発目的

新しいSNSの形として、”音声でコミュニケーションするSNS”をコンセプトとしたアプリケーションを新規開発する。

## 概要
> アプリの概要やジャンル

新しいSNSの形として、”音声でコミュニケーションするSNS”をコンセプトとしたアプリケーションを新規開発する。

## ターゲット
> 利用が想定される年齢層やターゲット

ターゲットはBeReal.を利用する高校生から~20代（いわゆる、Z世代）をターゲットとする。
BeReal.が人気の理由は以下であると分析する。
- そのままのリアルをシェアする
  - 映えない
  - 盛れない
  - 人気を測る数値の撤廃によるSNS疲れ防止
- 新しいSNSの形
  - ランダムなタイミングで投稿

以上のように、
SNS疲れしがちなユーザーにとって、ある程度機能が限定的であり、加工したり、フォロー・フォロワーという概念を意識しないSNSの需要が高いと分析する。また、斬新なコンセプトのSNSに需要があると分析する。

## システム要件
> 必要な機能の一覧
### 機能要件
> システムがどのような情報を管理したり処理したりすればよいか
### 非機能要件
> システムの裏側を決める

| MoSCoW分析項目        | 機能要件                                                                           | 非機能要件          | 
| :-------------------- | :--------------------------------------------------------------------------------- | :------------------ | 
| Must（対応必須）      | ログイン<br>新規登録<br>プロフィール画像設定<br>録音<br>送信<br>宛先指定<br>再生<br>通知   | 3,000ユーザー / 1日 | 
| Should（対応すべき）  | パスワード変更<br>再録音                                                           |                     | 
| Could（できれば対応） | ダークモード<br>プロフィール画像変更<br>ユーザー名変更<br>ウィジェット対応<br>返信 | FirebaseAnalytics   | 
| Won't（対応不要）     | フォロー・フォロワー管理<br>いいね機能                                             |                     |
